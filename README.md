# 動画重複削除システム
<img width="1322" height="619" alt="image" src="https://github.com/user-attachments/assets/981815d7-63e7-4351-9028-4c5212208e1d" />


指定したフォルダ内の重複動画ファイルを検出し、削除（ゴミ箱へ移動）するためのWebアプリケーションです。

## 機能
- 複数フォルダ（サブフォルダ含む）を同時にスキャン対象に指定可能
- **2種類の重複検出**
  - バイナリ一致: ファイルサイズ + 部分ハッシュ（先頭/中間/末尾16KB）の MD5 で完全一致を検出
  - サムネイル類似: 動画から抽出した代表フレームの pHash（DCT-II ベース）が近いものをグループ化（エンコード違いの同一動画も検出）
- 各ファイルの **サイズ・動画の長さ・サムネイル静止画** を一覧表示
- 選択したファイルの削除（ゴミ箱への移動）

## セットアップと実行

### 前提
- Node.js 18+
- システムに `ffprobe` / `ffmpeg` がインストールされていること
  - macOS: `brew install ffmpeg`
  - Linux: `apt install ffmpeg` など
  - ※ `ffmpeg-static` はバンドルしていますが、duration 取得に使う `ffprobe` バイナリは含まれないためシステム側に必要です。

### 手順

1. 依存関係のインストール:
   ```bash
   npm install
   ```
   `sharp` はネイティブビルドを伴います。失敗する場合は `npm rebuild sharp` をお試しください。

2. サーバーの起動:
   ```bash
   npm start
   ```

3. ブラウザでアクセス:
   [http://localhost:3000](http://localhost:3000)

## 仕組み
- スキャン時に各動画から `duration * 0.1` 秒地点の 1 フレームを ffmpeg で JPG として抽出（一時ディレクトリは `os.tmpdir()/video-dup-thumbs`）。
- 抽出したサムネイルを sharp で 32×32 グレースケール化し、2D DCT-II → 左上 8×8 を中央値ベースで 64bit ハッシュ化。
- pHash 同士のハミング距離が閾値（既定 5）以下のものを Union-Find で連結成分にまとめ、サムネイル類似グループとして提示。
- バイナリ一致グループに完全包含される視覚クラスタは抑制し、二重表示を防止。

## 設定
環境変数で調整可能な項目:
- `FFMPEG_CONCURRENCY` — ffmpeg/sharp の並列実行数（既定 3）

ソース内定数（`server.js`）:
- `PHASH_HAMMING_THRESHOLD` — サムネイル類似と判定するハミング距離の上限（既定 5）

## 注意事項
- 動画ファイルはサイズが大きい傾向があるため、バイナリ判定はファイルの先頭・中間・末尾の部分データのみで MD5 を計算しています。
- 削除されたファイルはシステムのゴミ箱に移動されますが、念のため重要なファイルはバックアップを取ってから操作してください。
- メタデータ（pHash・duration・サムネイル）はプロセス内のメモリにキャッシュされます。同一プロセス中にファイルを上書きした場合、古い値が返ることがあります。
- 数千件規模では pHash クラスタリングが O(n²) のため処理時間が伸びます。数百件規模を想定しています。
